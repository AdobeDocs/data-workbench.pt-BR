# Glossário da análise de big data{#data-workbench-glossary}

**Alerta**

Na análise de big data, uma mensagem ou um relatório que é automaticamente acionado quando uma métrica atinge um limite definido dentro de um intervalo de tempo especificado. Por exemplo, é possível criar um alerta que emite um email se o número de exibições de página durante um período de 24 horas cair acima ou abaixo de uma quantidade especificada.

**Analista**

Um indivíduo que realiza análises, define relatórios ou, de outra forma, usa análise de big data.

**Arquiteto**

Um indivíduo que determina como os dados serão capturados, processados e organizados para análise e relatório. Esse indivíduo geralmente tem uma experiência significativa na configuração da plataforma Adobe® para uso pelos analistas.

**Gráfico de barras**

Uma forma de gráfico que usa barras retangulares de vários tamanhos para mostrar comparações entre dois ou mais itens.

**Benchmark**

Uma medida ou norma que serve como ponto de referência pelo qual outras pessoas podem ser medidas ou julgadas. Na análise de big data, um benchmark ilustra a comparação entre o valor associado a uma seleção (um subconjunto selecionado) e o valor que seria mostrado se a seleção não fosse feita.

**Cardinalidade**

O número de elementos em um conjunto. O número de elementos em uma dimensão de dados da Adobe é chamado de cardinalidade da dimensão.

**Ponto de Verificação**

A hora em que uma cópia de um conjunto de dados da Adobe foi gravada para fins de backup ou recuperação. O termo também se refere ao conjunto de dados gravados durante uma operação de ponto de verificação.

**Qui-quadrado**

Um teste estatístico para determinar a probabilidade de um desvio observado do evento ou resultado esperado ocorrer por acaso. Na análise de big data, quanto mais próximo o quadrado Chi estiver de 100%, menor será a probabilidade de o desvio ocorrer apenas por acaso.

**Clickstream**

Um termo informal que se refere à sequência de páginas que um usuário solicita ao navegar em um site. As informações de sequência de cliques podem ajudar os proprietários do site a entender como os visitantes usam seus sites e quais páginas eles solicitam com mais frequência. Os proprietários de sites podem capturar os fluxos de cliques de visitantes para seus sites, mas não podem capturar fluxos de cliques fora do site (exceto usando cookies de terceiros ou tags de página) porque essas solicitações são registradas por servidores da Web externos.

**Legenda de cor**

Uma legenda na análise de big data que exibe o painel de cores de uma métrica selecionada. As legendas coloridas permitem a codificação de cores de visualizações por várias métricas. As visualizações de codificação de cores facilitam a detecção de anomalias, exceções e tendências.

**Painel de cores**

Em uma legenda de cor, o intervalo de cores usado para representar o intervalo de valores possíveis para uma métrica. Quando uma legenda de cor é aplicada a uma visualização, os elementos gráficos na visualização (por exemplo, as barras em um gráfico de barras) são codificados por cores de acordo com os valores representados pelo painel de cores.

**Chave comum**

Uma variável comum que forma uma relação entre as linhas em dois conjuntos tabulares de dados. Por exemplo, uma ID de produto pode agir como a chave comum entre uma tabela de variáveis de sequência de consulta e uma tabela de dados de produto de um sistema de gerenciamento de inventário.

**Intervalo de confiança**

Um intervalo de valores que tem uma probabilidade específica de conter a taxa ou tendência. Os intervalos de confiança de 80% (valor p = 0,20), 95% (valor p = 0,05) e 99% (valor p = 0,01) são os intervalos mais usados. (Fonte: `http://www.nci.nih.gov/statistics/glossary`)

**Nível de confiança**

A probabilidade de que o erro de amostragem num resultado de pesquisa caiba num intervalo especificado, normalmente expresso em termos de erros padrão (por exemplo, 1 erro padrão é igual a 68% de probabilidade, 2 erros padrão é igual a 95,4% de probabilidade). (Fonte: `http://www.magazine.org/research/3410.cfm`)

**Mapa de conversão**

Na análise de big data, um tipo de visualização em que os elementos são representados no eixo x de acordo com o valor da métrica de Conversão.

**Índice de conversão**

Na análise de big data, a porcentagem de sessões durante as quais um evento de valor ocorreu. A taxa de conversão é calculada dividindo o número de sessões em que um evento de valor ocorreu pelo número total de sessões.

**Correlação**

Uma medida numérica da intensidade de uma relação linear entre duas variáveis aleatórias. As variáveis que tendem a se mover juntas para cima ou para baixo estão positivamente correlacionadas, enquanto as variáveis que tendem a se mover em direções opostas estão negativamente correlacionadas. Consulte também Coeficiente de correlação.

**Coeficiente de correlação**

O valor numérico que significa a força de uma relação linear entre duas variáveis aleatórias. Consulte também Correlação.

**Dimensão contável**

Uma dimensão na qual o número de elementos que a dimensão contém pode ser contado discretamente. As dimensões contáveis podem ter dimensões filhas dos seguintes tipos: Contável, Numérico, Simples, Muitos para Muitos e Denormal.

**Transformação entre linhas**

No servidor da análise de big data, uma transformação de dados que permite que dados de vários registros de eventos para um Visitante (feito em momentos diferentes) sejam incorporados em um cálculo.

**Crosstab**

Na análise de big data, uma visualização tabular que exibe as métricas associadas às interseções de duas dimensões.

**Cubo**

Uma estrutura de dados multidimensional ou um grupo de células de dados organizados pelas dimensões dos dados. Por exemplo, uma planilha exemplifica uma matriz bidimensional com dados organizados em linhas e colunas. Em uma planilha, cada linha ou coluna é uma dimensão. Uma matriz tridimensional pode ser visualizada como um cubo com cada dimensão formando um lado do cubo, incluindo qualquer fatia paralela a esse lado. As matrizes de dimensões mais altas não têm metáfora física, mas podem ser usadas para organizar dados de forma que os usuários pensem sobre sua empresa. Também conhecido como hipercubo, matriz multidimensional ou banco de dados multidimensional.

**Painel**

Um espaço de trabalho que é criado para exibição em vez de interação. Os painéis fornecem status &quot;instantâneo&quot; por meio da exibição de indicadores chave de desempenho adequados para um gerente ou operador específico que esteja supervisionando um ou mais objetivos de negócios. Consulte também Espaço de trabalho.

**Extração de dados**

A aplicação não guiada ou interativa de uma coleção de procedimentos matemáticos aos dados históricos de negócio, numa tentativa de encontrar informações na forma de correlações e outras relações estatísticas.

**Subconjunto de dados**

Um recurso que permite aos usuários selecionar facilmente um subconjunto de um conjunto de dados para uso em análises online ou offline. (Um subconjunto de dados é uma parte do conjunto de dados com base em um filtro.)

**Data Warehouse**

Um banco de dados projetado para apoiar a tomada de decisões em organizações. Geralmente, um data warehouse contém grandes quantidades de dados não voláteis, orientados por assunto e variáveis, estruturados para consultas rápidas e online e resumos de gerenciamento.

**Unidade de processamento de dados**

Um tipo de servidor de análise de big data que processa, armazena e fornece dados de um conjunto de dados da Adobe. Como opção, uma DPU pode armazenar os arquivos de log VSL que contêm os dados de origem a partir dos quais o conjunto de dados é construído ou pode receber esses dados de uma unidade de servidor de arquivos (FSU) do servidor de análise de big data. Uma DPU é o tipo de servidor de análise de big data com o qual os clientes de análise de big data e de relatório® interagem diretamente.

**Conjunto de dados**

Os dados carregados e processados pelo servidor da análise de big data. O conjunto de dados representa os dados que podem ser transmitidos para análise de big data ou Relatório para fins de análise, relatório e alerta. Fisicamente, o conjunto de dados reside no arquivo temp.db. Cada computador do servidor de análise de big data (ou cluster do servidor de análise de big data) mantém um conjunto de dados.

**Dados do conjunto de dados**

Os dados criados e armazenados em um conjunto de dados do servidor de análise de big data. Inclui os dados de eventos e os dados de integração que são admitidos ou criados no conjunto de dados. Também inclui quaisquer informações derivadas desses dados, conforme determinado pelos arquivos de configuração que definem esse conjunto de dados. Os dados do conjunto de dados podem ser recriados ao reprocessar os dados do evento e da integração usando os mesmos arquivos de configuração ou arquivos de configuração diferentes. (Os arquivos de configuração são arquivos do sistema gerenciados como parte de um perfil da Adobe.)

**Registros do conjunto de dados**

Os registros de dados de eventos admitidos em um conjunto de dados da Adobe após toda a filtragem e outro processamento terem sido executados. Também chamadas de entradas de log processadas.

**Esquema de conjunto de dados**

Uma visualização na análise de big data que exibe o esquema do conjunto de dados que faz o backup do perfil atualmente selecionado.

**Espaço de Armazenamento do Conjunto de Dados**

A quantidade de dados (em bytes) inseridos ou criados em um conjunto de dados da Adobe. Esses dados são armazenados pelo conjunto de dados em um arquivo chamado Temp.db em um computador servidor de análise de big data. Os dados em Temp.db são transitórios e podem ser recriados pelo reprocessamento dos dados de origem (ou seja, dados de eventos e dados de integração) com os arquivos de configuração apropriados.

**Decodificador**

O componente no servidor da análise de big data que lê dados de eventos de várias fontes e gera dados usados para produzir o conjunto de dados. A saída de um decodificador pode ser usada como entrada para qualquer recurso de &quot;processamento de log&quot; no servidor de análise de big data. Os tipos de decodificador incluem o decodificador de sensor (para carregar dados de várias versões do sensor), o decodificador de Expressão regular (para carregar dados de arquivos simples delimitados) e o decodificador de ODBC (para carregar dados de fontes de dados ODBC).

**Dimensão**

Um conjunto de elementos, todos com um tipo semelhante na percepção do usuário. Os elementos definem um conjunto de categorias no qual os dados podem ser agrupados. Por exemplo, os elementos segunda, terça, quarta, quinta, sexta, sábado e domingo compõem uma dimensão de &quot;Dia da semana&quot;.

**Elemento de dimensão**

Uma categoria individual dentro de uma dimensão. Por exemplo, uma dimensão &quot;Dia da semana&quot; conteria os elementos individuais segunda, terça, quarta, quinta, sexta, sábado e domingo.

**Legenda da dimensão**

Na análise de big data, a legenda que lista as dimensões que foram definidas (ou derivadas) no conjunto de dados. Quando uma seleção é feita em uma visualização, a legenda Dimensão identifica as dimensões cujos valores diferem do benchmark por uma quantidade estatisticamente significativa.

**Drill-Up-Down**

Uma técnica analítica específica pela qual o usuário navega entre níveis de dados que vão do mais resumido (para cima) ao mais detalhado (para baixo). Por exemplo, ao exibir dados de vendas para a América do Norte, uma operação de detalhamento na dimensão Região pode exibir Canadá, Estados Unidos e México. Uma nova análise do Canadá pode exibir Toronto, Vancouver, Montreal e assim por diante.

**Dados do evento**

Os dados coletados pelos Sensores ou outros meios (por exemplo, um arquivo de log do servidor da Web), que constituem a entrada principal para o servidor da análise de big data. Cada registro de dados de evento representa um registro de transação ou uma única instância de um evento.

**Dimensão estendida**

Uma dimensão baseada em dados estendidos. Dados estendidos são quaisquer dados além do mínimo necessário para formar um registro de dados de evento válido. Os dados estendidos podem ser adicionados a um registro de dados de evento quando o evento original é capturado ou pode ser incorporado de outras fontes e adicionado ao registro de dados de evento como dados de integração. Qualquer dimensão que se baseie nestes dados adicionais é considerada &quot;alargada&quot;.

**Unidade de servidor de arquivos (FSU)**

Um tipo de servidor de análise de big data cuja função é exclusivamente receber dados de eventos de um ou mais servidores Sensores ou repetidores e fornecer dados a uma ou mais Unidades de Processamento de Dados (DPUs) do servidor de análise de big data para uso na construção de conjuntos de dados da Adobe. Os FSUs otimizam a transferência de dados de eventos para as DPUs e são significativamente mais rápidos que os servidores de arquivos comuns. O uso de um FSU reduz os custos de hardware, permitindo que os dados de log sejam armazenados em hardware de armazenamento de menor custo e reduz a complexidade administrativa, permitindo que vários Sensores apontem para um único servidor de análise de big data.

**Avaliação de consulta incremental**

O processo patenteado pelo qual o servidor de análise de big data fornece resultados de consulta imediatos a um usuário com base em uma amostra aleatória projetada da população completa. Sob esse processo, o servidor refina gradualmente a precisão da consulta considerando mais dados até que todos os dados tenham sido considerados e uma contagem exata tenha sido obtida.

**Dados de integração**

Os dados de integração são dados externos de bancos de dados corporativos ou arquivos de pesquisa que podem ser combinados com dados de eventos para criar o conjunto de dados. Em geral, você usa dados de integração para aumentar os dados de evento adquiridos pelo Sensor. (Conceitualmente, você pode pensar em usar dados de integração para preencher registros de dados de eventos com colunas adicionais de informações.)

**Legenda**

Uma janela na análise de big data que fornece detalhes explicativos sobre as visualizações exibidas na área de trabalho. Os tipos de legendas incluem legendas de cor, legendas de dimensão e legendas de métrica. Como qualquer janela na análise de big data, as janelas de legenda podem ser geradas e distribuídas pelo Relatório.

**Gráfico de linhas**

Um tipo de visualização na análise de big data que representa métricas para uma dimensão especificada como pontos sucessivos no eixo x de um gráfico e, em seguida, conecta os pontos com linhas. Um gráfico de linha é uma maneira particularmente eficaz de visualizar uma métrica em uma dimensão baseada em tempo.

**Condição de entrada do registro**

Uma condição que determina se um registro de dados de evento (uma entrada de log) será incluído no conjunto de dados. Por exemplo, uma condição de entrada de log pode especificar que somente os registros de dados de eventos associados a um site específico sejam admitidos no conjunto de dados. As condições de entrada do log são especificadas no arquivo de configuração de processamento do log no servidor da análise de big data.

**Dimensão de muitos para muitos**

Na plataforma Adobe, uma dimensão que tem uma relação muitos para muitos com uma dimensão pai Contável. Uma dimensão muitas para muitas representa um conjunto de valores para cada elemento de sua dimensão pai. Por exemplo, no Site, a dimensão Frase de pesquisa tem uma relação muitos para muitos com seu pai, a dimensão Sessão (ou seja, uma Sessão pode ter qualquer número de Frases de pesquisa e uma Frase de pesquisa pode ter qualquer número de Sessões).

**Mascaramento**

Um recurso na análise de big data que permite que os analistas ocultem temporariamente elementos que não desejam incluir em uma análise.

**Média**

A média aritmética de um conjunto de números. A soma dos dados dividida pelo tamanho da amostra.

**Medianiz**

Um número que separa a metade mais alta de uma amostra, uma população ou uma distribuição de probabilidade da metade mais baixa. Metade da população terá valores inferiores ou iguais à mediana e metade da população terá valores iguais ou superiores à mediana.

**Métrica**

Na Adobe, uma fórmula nomeada que descreve como calcular um valor quantitativo a partir dos dados no conjunto de dados. No Site, por exemplo, a métrica &quot;Sessões por visitante&quot; representa uma fórmula que divide a contagem de Sessões pela contagem de Visitantes.

**Legenda da métrica**

Uma janela na análise de big data que exibe as métricas definidas pelo perfil ativo. Uma legenda de Métrica exibe o valor de cada métrica, como calculado a partir do conjunto de dados ou da seleção atual (se uma seleção estiver ativa no espaço de trabalho, a legenda de Métrica exibirá os valores do subconjunto selecionado em vez do conjunto de dados inteiro). Como qualquer janela na análise de big data, as legendas de métricas podem ser geradas e distribuídas pelo Relatório.

**Planilha de métricas**

Uma janela na análise de big data que permite aos analistas definir suas próprias métricas para um conjunto de dados. Uma planilha Métrica é semelhante a uma planilha. Usando a sintaxe de fórmula da análise de big data, os analistas podem inserir expressões que descrevem os valores quantitativos que desejam derivar do conjunto de dados. Por exemplo, um analista pode definir uma métrica que exibe a porcentagem de Visitantes que visualizaram uma página de um domínio específico. Como as métricas regulares, as fórmulas em uma planilha Métrica operam no subconjunto selecionado quando uma seleção está ativa no espaço de trabalho. Como qualquer janela na análise de big data, as planilhas de métricas podem ser geradas e distribuídas pelo Relatório.

**Nova condição do visitante**

A condição que determina se uma nova ID de rastreamento é criada quando o servidor da análise de big data é apresentado com um registro de dados de evento.

**Nó**

Um agrupamento de um ou mais itens discretos em uma única entidade lógica. Em um arquivo de configuração (.cfg), um nó é um item que contém parâmetros relacionados. Consulte também Parâmetro e Vetor. Na análise de big data, um nó em um Mapa de processos representa uma única página ou um grupo definido de páginas.

**Dimensão numérica**

Na plataforma Adobe, uma dimensão que possui valores ordenados e numéricos e uma relação um para muitos com uma dimensão pai Contável. Geralmente, uma dimensão Numérica representa uma propriedade numérica dos elementos da dimensão pai. Normalmente as dimensões numéricas são usadas para definir métricas de “soma”.

**Relação Um para muitos**

Uma relação entre duas dimensões de dados na qual um único elemento em uma dimensão está (ou pode ser) relacionado a um ou mais elementos na outra dimensão.

**Ordinal**

Ser ou denotar uma ordem numérica em uma série. (Fonte: `http://wordnet.princeton.edu/perl/webwn?s=ordinal`) Se uma dimensão for classificada normalmente na análise de big data, os elementos da dimensão serão exibidos na ordem em que são representados internamente.

**Fora**

Em um conjunto de dados, um valor até agora removido de outros valores na distribuição que sua presença não pode ser atribuída à combinação aleatória de causas de chance.

**Sobreposição de página**

Um tipo de visualização na análise de big data que codifica os links em cores em uma imagem de uma página da Web de acordo com uma métrica especificada. Você pode usar uma visualização Sobreposição de página para identificar rapidamente quais links em uma página chamam a atenção dos visitantes (e levá-los para outras páginas do site) e quais não. Você também pode usá-lo para entender o &quot;valor&quot; (medido pelos eventos de valor) que os vários links em uma página geram para o seu site.

**Condição de exibição de página**

Uma opção em uma transformação de servidor da análise de big data que permite que os registros de eventos sejam incluídos ou excluídos como exibições de página com base no tipo de conteúdo ou no conteúdo real. Você pode usar essa opção, por exemplo, para excluir registros de eventos pertencentes a solicitações HTTP malsucedidas (por exemplo, aquelas que produziram um código de status 404) ou solicitações que retornam certos tipos de conteúdo (por exemplo, solicitações de imagem). Se um registro de evento HTTP não for filtrado pela opção Condição de exibição de página, esse registro de evento representará uma exibição de página no conjunto de dados.

**Visualização do navegador de caminhos**

Um tipo de visualização na análise de big data que permite que um analista explore interativamente sequências de eventos (como Exibições de página) em uma Sessão de visitante ou em Sessões de visitante.

**Servidor primário**

Em um cluster de servidores de análise de big data, o servidor de análise de big data que intermedia as comunicações entre clientes (como análise de big data e Relatório) e os outros servidores no cluster. O servidor primário também funciona como o ponto focal administrativo para o cluster. Usando os recursos de sincronização de perfil do servidor de análise de big data, as alterações feitas por um administrador no servidor primário são propagadas automaticamente para os outros servidores no cluster. Um servidor primário é uma DPU de servidor de análise de big data.

**Mapa de processos**

Um tipo de visualização na análise de big data que permite ao analista entender o fluxo de tráfego entre páginas ou nós em um site. Um mapa de processos exibe informações sobre páginas específicas (como o número de sessões durante as quais as páginas foram visualizadas) e também ilustra o volume de tráfego entre páginas ou nós.

**Servidor de processamento**

Em um cluster de servidores de análise de big data, os servidores de análise de big data são controlados pelas informações de configuração no servidor primário. Um servidor de processamento processa dados de eventos em um conjunto de dados e responde a consultas de clientes, como análise de big data e Relatório. Ao responder a uma consulta, um servidor de processamento divide (&quot;partições&quot;) a responsabilidade de realizar a consulta entre os servidores no cluster. Quando os outros servidores concluírem essas partes da consulta, o servidor de processamento combinará (&quot;partições&quot;) seus resultados e retornará o resultado combinado ao cliente.

**Perfil**

Um conjunto de arquivos de configuração que contém as regras para a criação de um conjunto de dados para uma finalidade de análise específica. Um perfil também define artigos como métricas, dimensões derivadas, espaços de trabalho, relatórios, visualizações e legendas que permitem que os analistas interajam com o conjunto de dados e obtenham informações dele. Um perfil pode ser estruturado geralmente para um site da Web (como em um perfil para www.mysite.com) ou pode ser personalizado para um tipo específico de usuário (como o perfil de usuário &quot;Marketing&quot; ou &quot;Finanças&quot;).

**Gerenciador de perfil**

Uma ferramenta administrativa interativa na análise de big data que permite que um administrador ou outro usuário gerencie os arquivos de configuração associados a um perfil.

**Expressão regular**

Uma fórmula que descreve ou corresponde a um conjunto de strings de acordo com determinadas regras de sintaxe. Expressões regulares (normalmente abreviadas como regexp, regex ou regxp) são usadas para pesquisar e manipular corpos de texto com base em determinados padrões. A notação de expressão regular teve origem nos primeiros editores do Unix e ganhou uso amplo em outros utilitários do Unix, como vi e Perl. Atualmente, expressões regulares são suportadas por muitos editores de texto, linguagens de script e outras ferramentas de manipulação de texto. O servidor da análise de big data inclui um mecanismo de expressão regular.

**Mapa de retenção**

Um tipo de visualização na análise de big data que representa os elementos no eixo x de acordo com o valor da métrica Retenção.

**Gráfico de Dispersão**

Um tipo de visualização na análise de big data que representa dados de duas variações como pontos em um gráfico. Um gráfico de dispersão tem dois dados para cada elemento que está sendo grafado. Por exemplo, um gráfico de dispersão das métricas Taxa de conversão e Visitantes para um conjunto de 10 páginas resultaria em 10 pontos separados no gráfico.

**Seleção**

Um recurso na análise de big data que permite que um analista restrinja o conjunto de dados recuperados e exibidos para relatórios ou análises. Uma seleção pode ser feita interativamente na análise de big data clicando-se em elementos exibidos em uma ou mais dimensões (o clique com o botão esquerdo seleciona um elemento, o clique com o botão direito cancela a seleção de um elemento). As seleções também podem ser feitas através da definição de filtros que selecionam certos elementos em dimensões especificadas.

**Dimensão simples**

Na plataforma Adobe, uma dimensão que tem uma relação um para muitos com uma dimensão pai contável. Por exemplo, Referenciador de visitantes é uma dimensão simples cujo pai é a dimensão contável, Visitante. Um Visitante tem apenas um Referenciador de Visitantes. No entanto, um Referenciador de visitantes pode ter vários Visitantes (ou seja, um Referenciador de visitantes pode estar relacionado a muitos Visitantes).

**Suavização**

Uma inferência matemática de uma curva em vários pontos em um gráfico de linhas, que é usada para ilustrar uma linha de tendência mais significativa entre pontos de dados relativamente esparsos.

**Fonte**

No servidor de análise de big data, um recurso que contém dados de evento que podem ser usados para criar um conjunto de dados. Os decodificadores da Adobe procuram por fontes dados de eventos para decodificar para uso pelo servidor de análise de big data.

**Dados de origem**

No servidor de análise de big data, os dados que são a entrada para um de seus decodificadores. Os dados de origem podem ser inseridos no decodificador de Sensor (que carrega dados de origem de várias versões do Sensor), no decodificador de Expressão Regular (que carrega dados de origem de arquivos simples delimitados) e no decodificador ODBC (que carrega dados de origem de fontes de dados ODBC).

**Gráfico de tabela**

Um tipo de visualização na análise de big data que exibe dados em formato tabular. Os valores de métrica em um gráfico de tabela podem ser expressos numericamente ou representados em formato de barra.

**Séries cronológicas**

Um gráfico que mostra como determinada propriedade ou valor muda ao longo do tempo.

**ID de rastreamento**

Um identificador que diferencia exclusivamente as entidades primárias que estão sendo analisadas em um conjunto de dados da Adobe. Uma ID de rastreamento pode ser construída de várias fontes, incluindo uma ID exclusiva de um cookie cliente Web, um número IP e um hash agente-usuário ou um nome x.509. Embora essas sejam algumas fontes comuns para uma ID de rastreamento, qualquer valor capaz de identificar exclusivamente as entidades que preenchem a dimensão Visitante no conjunto de dados pode ser usado.

**Transformação**

Um método para alterar o valor de uma variável usando alguma operação matemática. No servidor da análise de big data, por exemplo, os analistas podem usar a transformação de divisão para dividir os pares nome-valor em uma sequência de consulta em variáveis individuais.

**Gráfico de barras bicidimensional**

Um tipo de visualização na análise de big data que exibe simultaneamente duas dimensões e até duas métricas em uma exibição gráfica tridimensional.

**Legenda de valor**

Uma janela na análise de big data que permite ao analista associar um valor monetário a um evento selecionado e exibir os resultados quando esse valor monetário é somado para todas as instâncias do evento selecionado no conjunto de dados.

**Visitante**

A dimensão no conjunto de dados que identifica a entidade que gerou o evento. Cada membro da dimensão Visitante é identificado por uma ID de rastreamento exclusiva. No Site, por exemplo, a ID de rastreamento normalmente é derivada de uma ID exclusiva dentro do cookie do cliente. Em Chamada, a ID de rastreamento pode ser o número de telefone do chamador.

**Referenciador do visitante**

O primeiro referenciador HTTP para um Visitante dentro do intervalo de tempo de um conjunto de dados da Adobe.

**Divisão do visitante**

Um recurso no servidor da análise de big data que permite que os visitantes com grandes quantidades de dados de eventos sejam divididos entre duas IDs de rastreamento. A divisão do visitante é usada para impedir que os dados do evento sejam filtrados do conjunto de dados quando um visitante excede o número máximo configurado de eventos por visitante (um parâmetro definido para garantir o desempenho correto do sistema). Embora a divisão do visitante aumente artificialmente o número de visitantes no conjunto de dados, ela não infla o número total de registros de eventos, o que garante que o número total de eventos contáveis (por exemplo, exibições de página, marcações) permaneça preciso.

**VSL**

Arquivo de log. O tipo de arquivo no qual o servidor da análise de big data armazena dados de evento que recebem do Sensor. Os arquivos VSL são compactados, mas podem ser exibidos na forma descompactada usando a funcionalidade de transformação de dados. VSL é a extensão de arquivo para um arquivo de log.

**Análise de Hipóteses**

Um tipo de análise usado para entender as consequências da alteração de determinadas variáveis em um modelo de dados, observando o efeito que as alterações nessas variáveis exercem sobre outros dados no modelo.

**Workspace**

Na análise de big data, um espaço de trabalho é um contêiner para uma tarefa de análise e visualização específica. Um espaço de trabalho pode conter várias visualizações), todas operando em relação a um conjunto comum de dados (ou seja, todas as visualizações no espaço de trabalho renderizam o mesmo conjunto de resultados de consulta). Quando um analista executa uma operação de subdefinição ou filtragem em uma visualização, o subconjunto selecionado é refletido em toda a área de trabalho.

**Desktop**

A área &quot;inicial&quot; na interface do usuário da análise de big data que permite organizar e acessar todos os seus espaços de trabalho e relatórios. Ela também permite que você crie e salve espaços de trabalho e relatórios novos e atualizados no servidor da análise de big data para que outras pessoas que usam o mesmo perfil possam acessá-los também.
